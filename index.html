🟠 RFQ / 規劃（M-18 ~ M-12）

需求確認與資源排程

📋 PJM 核心任務

  • ▸確認客戶規格書 (Customer Spec) 與訂單意向書 (LOI)
  • ▸規劃初版 BOM 架構與目標成本 (COG Target)
  • ▸ODM/JDM 廠商選定與 NDA / 商務條款洽談
  • ▸制定跨部門 NPI 里程碑計畫表 (Master Schedule)

⚠️ 常見卡點

  • ▸客戶規格書版本反覆，BOM 架構難以確定
  • ▸多個並行專案搶人，關鍵 RD 人力無法及時到位
  • ▸ODM 廠報價週期過長，影響成本決策時程

🔴 高風險警示

規格未凍結即下單 Long Lead Time 零件，後續 Design Change 產生高額 ECO 成本

🟡 DVT — Design Verification Test（M-12 ~ M-8）

設計驗證與關鍵零件認証

📋 PJM 核心任務

  • ▸追蹤 EE / ME / Thermal 設計進度，主持每週 Issue Review 會議
  • ▸確認 GPU / CPU / HBM 等關鍵零件的 AVL 認証狀態
  • ▸下單 Long Lead Time 零件並建立交期追蹤看板
  • ▸制定 DVT 測試計畫，預約 SIT 實驗室資源

⚠️ 常見卡點

  • ▸GPU 等主晶片 Lead Time 超過 20 週，樣品到位嚴重延遲
  • ▸散熱設計不達標，需重新 Layout，延期 6~8 週
  • ▸EE 與 ME 設計邊界衝突，需多輪協調消耗大量時程

🔴 高風險警示

上游晶片廠（如 NVIDIA）臨時更新規格 (Spec Rev)，強制觸發 BOM 修改，牽動整條 Critical Path

🟢 EVT — Engineering Verification Test（M-8 ~ M-5）

BOM 凍結與工程全面驗證

📋 PJM 核心任務

  • ▸主導 BOM Freeze 會議，鎖定工程確認版本
  • ▸協調 FAE 排入燒機 (Burn-in) 與相容性測試
  • ▸追蹤 EVT Issue List，設定 Owner 與 ETA，強力推動收斂
  • ▸安排客戶 Pre-qualification 送樣與技術文件準備

⚠️ 常見卡點

  • ▸客戶臨時追加規格（Scope Creep），BOM Freeze 一再遞延
  • ▸測試 Fail 反覆，PCBA 需多輪重工，迭代週期拉長
  • ▸多個 Issue 相互依賴，形成 Critical Path 死結，需 PJM 強力拆解

🔴 高風險警示

BOM 未準時凍結，導致 PVT 備料失準，量產備料成本暴增且交期延誤

🔵 PVT — Production Verification Test（M-5 ~ M-2）

量產驗證與工廠就緒

📋 PJM 核心任務

  • ▸確認工廠產線 SOP、治具設計與測試設備準備就緒
  • ▸協調 DFM（Design for Manufacturing）審查，確認製造可行性
  • ▸追蹤良率（Yield Rate）達標進度，設定 Go / No-Go 門檻
  • ▸準備客戶驗廠（Factory Audit）所需文件與現場安排

⚠️ 常見卡點

  • ▸良率低於目標（< 95%），良率改善需 2~4 週額外迭代
  • ▸工廠端產線換線排程衝突，PVT 批量入廠遭延遲
  • ▸治具（Jig / Fixture）製作週期過長，卡住測試整體節奏

🔴 高風險警示

良率改善方案若影響 PCB 設計，需重走 EVT 部分驗證流程，回流成本極高

🟣 MP — Mass Production（M-0 起）

量產 Ramp 與供應鏈管理

📋 PJM 核心任務

  • ▸管理首批量產出貨（Ramp-up）節奏與客戶交期承諾
  • ▸即時監控 Field Return Rate 與工廠 DPPM（Defect PPM）
  • ▸協調缺料（Short Supply）應急替代料（Alternate Part）認証方案
  • ▸維護長期供應鏈健康度指標與 EOL（End-of-Life）預警管理

⚠️ 常見卡點

  • ▸Ramp 速度不及預期，無法如期達到客戶要求的出貨量
  • ▸關鍵元件突發性缺料，影響整條生產線的排程
  • ▸Field Issue 爆發，需緊急啟動 8D Problem-solving 流程應對客訴

🔴 高風險警示

Ramp 延遲直接縮小與競品的時間差，市佔搶佔速度受損，商業損失難以量化

【過去】軟體產品開發

Sprint 迭代 / Release

【現在】雲端伺服器企劃

Roadmap ➔ NPI ➔ MP

擬定符合市場需求的產品規格發展，並與 RD 合作掌控 NPI 研發時程。

2. 跨職能溝通與利害關係人

【過去】軟體跨部門協作

UI/UX ➔ Frontend ➔ Backend

對齊設計師與工程師的認知，確保軟體介面與系統邏輯一致。

【現在】硬體系統整合

EE / ME / Thermal / 工廠

模組開發 / 海內外產能配置
